# WG Hope Bot v1.3.1

Фокус релиза: исправление обнуления `RX/TX всего` в веб-мониторинге после перезагрузки сервера.

## Кратко

После reboot ядро/интерфейсы сбрасывают сырые счётчики трафика. Из-за этого в мониторинге поля `RX/TX всего` могли уходить в ноль или резко уменьшаться.  
В `v1.3.1` totals переведены на накопительную модель хранения, которая не зависит от reset сырых счётчиков.

## Что изменено

- Исправление для пользователей/дашборда:
  - добавлена таблица `wg_peer_totals`;
  - totals по peer-ам теперь накапливаются инкрементально в `collect_once()`;
  - при первом запуске выполняется one-time backfill totals из исторических `wg_peer_samples`.

- Исправление для вкладки `Серверы`:
  - добавлена таблица `uplink_totals`;
  - totals по uplink-интерфейсам теперь накапливаются в `collect_uplink_once()`;
  - добавлен one-time backfill totals из `uplink_samples`.

- Обновлены точки чтения данных:
  - `load_dashboard_data()` и `load_user_realtime()` используют `wg_peer_totals` для `RX/TX всего`;
  - `load_servers_data()` и `load_servers_realtime()` используют `uplink_totals` для `RX/TX всего`.

## Практический эффект

- После reboot:
  - `RX/TX всего` на главной не обнуляются;
  - `RX/TX всего` в профилях пользователей не обнуляются;
  - `RX/TX всего` во вкладке `Серверы` не обнуляются.
- Реалтайм-скорости и графики продолжают работать как раньше.
