# 🚀 WG Hope Bot

Привет! Я перепаковал проект и выпустил полноценный релиз с чистой архитектурой и удобной админкой.

## 📌 Релизы

- `v0.0.1` - историческая версия, которая была на GitHub до обновления
- `v1.0.0` - новый стабильный релиз с переработанным ботом

## ✨ Что теперь умеет бот (v1.0.0)

- 👤 Роли пользователей: `super_owner`, `admin`, `user`, `pending`, `banned`
- 🔐 Управление доступом: заявки, одобрение, бан
- 🧩 Создание/выдача/удаление WireGuard-конфигов
- 🖼️ Отправка `.conf` + QR-кода
- 📊 Статистика по пользователям и IP
- 📝 Логи действий админов и пользователей
- 📣 Рассылки по выбранным chat_id или всем одобренным
- 🎚️ Лимиты числа конфигов на пользователя

## 🧱 Структура проекта

- `vpn_bot/main.py` - запуск приложения
- `vpn_bot/handlers.py` - Telegram-сценарии и FSM-режимы
- `vpn_bot/actions.py` - операции с конфигами/файлами
- `vpn_bot/wireguard.py` - вызовы `wg/wg-quick`, IP-аллокатор
- `vpn_bot/db.py` - SQLite, роли, клиенты, логи, статистика
- `vpn_bot/keyboards.py` - кнопки и inline-меню
- `vpn_bot/settings.py` - конфигурация через переменные окружения

## ⚙️ Что нужно для работы

- Linux-сервер с установленными `wg` и `wg-quick`
- Доступ бота на запись в `WG_CONF` (обычно `/etc/wireguard/wg0.conf`)
- Настроенный WireGuard интерфейс

## 🧪 Что работает уже сейчас

- ✅ Telegram-часть: меню, роли, админка, логи, рассылки
- ✅ Хранение данных и миграции SQLite
- ✅ Генерация и отправка клиентских файлов
- ✅ Добавление/удаление peer через WireGuard команды

## ⚠️ Что важно знать

- Без установленного `wg` создание конфигов не сработает (это нормально)
- Секреты не хранятся в репозитории
- Используй только `.env` на сервере, а в git держим `.env.example`

## 🔒 Безопасность

В репозиторий не попадают:

- `.env` с токенами
- реальные ключи
- боевые конфиги клиентов
- базы данных и артефакты `clients/`

## ▶️ Быстрый запуск (локально)

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# заполни BOT_TOKEN, SUPER_OWNER_CHAT_ID, SERVER_ENDPOINT, SERVER_PUBLIC_KEY
./run.sh
```

## 🐳 Docker запуск

```bash
docker compose up -d --build
docker logs -f wg-bot
```
