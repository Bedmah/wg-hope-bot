# Changelog

## v1.2.3

- Усилен production-контур для динамических uplink интерфейсов и регионов:
  - добавлена принудительная нормализация `Table = off` при замене VPN-uplink конфигов, чтобы исключить перехват default route;
  - добавлена синхронизация uplink-сервисов с systemd при старте бота (`enable/start` для включённых интерфейсов, `stop/disable` для выключенных);
  - включённые uplink-интерфейсы теперь поднимаются сразу и корректно переживают reboot.
- Улучшена безопасность маршрутизации и регионов:
  - маршрутизация больше не использует выключенные uplink интерфейсы (`enabled=0`);
  - назначение региона запрещено для выключенных интерфейсов и неготовых VPN-uplink;
  - проверка интерфейсов в админке дополнена stale-handshake probe (`probe=ok|fail`) для уменьшения ложных статусов.
- Улучшена надёжность мониторинга и восстановления:
  - улучшено поведение down-alert после рестарта для ранее упавших uplink (`UPLINK_ALERT_DOWN_ON_START`, включён по умолчанию);
  - исправлены переходы состояния алертов: recovery корректно фиксирует `last_alert_state=ok`.
- Консолидированы сетевые/production обновления:
  - дефолтная подсеть обновлена до `VPN_SUBNET=10.8.0.0/22` в настройках и шаблонах;
  - systemd-запуск бота обновлён на прямой вызов `bot.py` для более чистого старта.
- Результат по стабильности:
  - подтверждена устойчивость после повторных reboot: автоподъём uplink, сохранение policy routing и отсутствие нежелательного fallback в другой регион при доступном целевом uplink.

## v1.2.2

- Broadcast UX reworked in admin panel:
  - added recipient groups: `pending`, `approved`, `banned`, `all`;
  - added staged flow for broadcast target selection;
  - added manual chat_id additions on top of group selection.
- Broadcast delivery improved:
  - support for sending `text`, `photo`, `video`, `document`;
  - post-broadcast report now includes failed recipients with `chat_id`, `@username`, and error reason.
- Routing stability hardening:
  - cleanup of stale managed `iptables` rules before applying current state.
- Uplink/region persistence fixes:
  - removed automatic resurrection of deleted default regions/interfaces on each restart;
  - default region auto-recovery when none is marked in DB.
- Startup cleanup:
  - removed package-level import pattern causing `RuntimeWarning` about `vpn_bot.main` execution order.

## v1.2.1

- UX improvement for config creation in Telegram bot:
  - prompt changed to: "Enter config name" (RU locale string updated).
  - config name input step now shows a dedicated keyboard with Back button.
- Pressing Back now explicitly cancels config creation and returns to main menu.

## v1.2.0

- Добавлена региональная маршрутизация конфигов (`region` у клиентов) с выбором региона пользователем в боте.
- Добавлен policy routing по uplink-интерфейсам (`ip rule`/`rt_tables`/`iptables`) с синхронизацией через `sync_client_egress_routes()`.
- Добавлен автоматический failover:
  - при падении VPN uplink трафик регионов переводится на системный интерфейс (например `eth0`);
  - при восстановлении uplink маршрутизация возвращается автоматически.
- Добавлена админка серверов в Telegram-боте:
  - управление интерфейсами/регионами;
  - установка региона по умолчанию;
  - замена uplink-конфига;
  - удаление интерфейсов и регионов кнопками (inline) вместо ручного ввода.
- Добавлен мониторинг uplink-интерфейсов в боте:
  - healthcheck по интерфейсам;
  - уведомления админам о `DOWN/RECOVERY`;
  - параметры через `.env` (`UPLINK_*`).
- Существенно расширен веб-мониторинг:
  - новая вкладка `Серверы`;
  - статистика по регионам и интерфейсам;
  - графики скорости/ping/state, uptime, события состояния;
  - отображение региона в конфиге пользователя;
  - события смены региона в `Последние события` + фильтр `region_change`;
  - отображение `Время сервера` на всех страницах.
- Обновлена документация и шаблоны production-развёртывания:
  - `README.md`
  - `.env.example`
  - `deploy/systemd/*`
  - `deploy/scripts/install_prod_v1_2_0.sh`
