# Changelog

## v1.2.0

- Добавлена региональная маршрутизация конфигов (`region` у клиентов) с выбором региона пользователем в боте.
- Добавлен policy routing по uplink-интерфейсам (`ip rule`/`rt_tables`/`iptables`) с синхронизацией через `sync_client_egress_routes()`.
- Добавлен автоматический failover:
  - при падении VPN uplink трафик регионов переводится на системный интерфейс (например `eth0`);
  - при восстановлении uplink маршрутизация возвращается автоматически.
- Добавлена админка серверов в Telegram-боте:
  - управление интерфейсами/регионами;
  - установка региона по умолчанию;
  - замена uplink-конфига;
  - удаление интерфейсов и регионов кнопками (inline) вместо ручного ввода.
- Добавлен мониторинг uplink-интерфейсов в боте:
  - healthcheck по интерфейсам;
  - уведомления админам о `DOWN/RECOVERY`;
  - параметры через `.env` (`UPLINK_*`).
- Существенно расширен веб-мониторинг:
  - новая вкладка `Серверы`;
  - статистика по регионам и интерфейсам;
  - графики скорости/ping/state, uptime, события состояния;
  - отображение региона в конфиге пользователя;
  - события смены региона в `Последние события` + фильтр `region_change`;
  - отображение `Время сервера` на всех страницах.
- Обновлена документация и шаблоны production-развёртывания:
  - `README.md`
  - `.env.example`
  - `deploy/systemd/*`
  - `deploy/scripts/install_prod_v1_2_0.sh`

## v1.1.1

- Hotfix: исправлена битая кодировка русского текста в веб-мониторинге.
- Исправлены подписи в фильтрах мониторинга (периоды и режимы графиков).

- Улучшена читаемость интерфейса без изменения логики сбора/аналитики.

## v1.1.0

- Добавлена веб-панель мониторинга (`monitor/app.py`) с авторизацией, дашбордом и страницами пользователей.
- Добавлены графики по трафику/скорости/handshake с периодами: `1h`, `24h`, `7d`, `30d`, `all`.
- Добавлены фильтры мониторинга: по конфигу, типу событий, поиску и области отображения графиков.
- Добавлена пагинация и режим "Показать всё" для событий на странице пользователя.
- Добавлено хранение истории чатов пользователей в папке `chat/` (`chat/<chat_id>.log`) и выгрузка лог-файла из админки.
- Добавлена синхронизация профилей Telegram (username/first_name/last_name) из админки.
- Добавлена кастомизация текстов бота через админку: редактирование "Инструкция" и "Вопросы / Поддержка".
- Для `pending` пользователей добавлены кнопки "Проверить статус" и "Доступ / Вопросы".
- Улучшена админ-клавиатура и навигация по разделам.
- Улучшена работа с именами конфигов:
  - поддержка русского языка в отображаемом имени;
  - скрытие `chat_id` в отображаемых названиях и списках;
  - хранение `display_name` в meta-файле конфига.
- Исправлены проблемы:
  - "Изменить поддержку" в админке могла не срабатывать из-за пересечения обработчиков;
  - ложный успех при установке лимита несуществующему `chat_id`;
  - фильтрация событий по конфигу перенесена на SQL-уровень до `LIMIT`.
