# Release Notes v1.4.0

Дата: 2026-04-04

## Что обновили

- Прокачали стабильность мониторинга в production: снизили нагрузку на БД и уменьшили риск `database is locked` при одновременной активности бота и веб-морды.
- Оптимизировали realtime-обновления в интерфейсе мониторинга: polling стал реже и не грузит сервер, когда вкладка неактивна.

## Новое в «Серверах»

- Добавили корректные totals клиентского трафика по каждому интерфейсу (`RX/TX всего`).
- Добавили выбор периода для статистики интерфейсов:
  - `7 дней`
  - `30 дней`
  - `Всё время`
- Для каждого интерфейса теперь доступны:
  - `RX/TX` за выбранный период
  - `RX/TX всего` за всё время
- Добавлен суммарный блок по всем интерфейсам сразу.
- Добавлен блок архивных интерфейсов, чтобы исторические totals не терялись после удаления интерфейса.

## Важное исправление

- Исправлен подсчёт totals для `eth0`: теперь значения считаются по трафику клиентских конфигов, которые реально выходят через этот интерфейс.

## Результат

- Веб-мониторинг отвечает стабильнее под нагрузкой.
- Статистика по интерфейсам стала точнее, полнее и удобнее для сравнения по периодам.
- Удаление/изменение интерфейсов больше не «съедает» накопленные данные по трафику.
