# Release Notes v1.3.4

Дата: 2026-04-04

## Что исправили

- Убрали массовую перестройку маршрутизации при обычной смене региона у одного пользователя.
- Переключение региона теперь меняет только маршрут IP конкретного клиента.
- Защищены операции маршрутизации от гонок (lock), чтобы параллельные действия не ломали сетевое состояние.

## Что улучшили

- Повторный выбор уже активного региона теперь no-op и не вызывает лишних сетевых действий.
- Создание/удаление конфигов больше не запускает глобальный reconcile правил для всех клиентов.
- Глобальная синхронизация маршрутов сохранена для админских/topology-сценариев:
  - добавление/удаление интерфейса;
  - добавление/удаление региона;
  - смена default-региона;
  - изменения состояния uplink.

## Ожидаемый эффект

- При смене региона одним пользователем остальные клиенты не должны терять интернет из-за глобальной очистки `ip rule`/`iptables`.
- Поведение стало более предсказуемым и стабильным при росте числа пользователей и регионов.
