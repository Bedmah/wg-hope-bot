# WG Hope Bot v1.3.2

Фокус релиза: исправление сценария рассылок в админке и завершение UX-контура для заблокированных пользователей.

## Кратко

В релизе закрыты две прикладные проблемы из production:

1. В `Админка -> Сообщения` кнопки групп иногда сбрасывали диалог в `Меню` вместо продолжения шага рассылки.
2. У роли `banned` отображалось обычное меню, но действия были недоступны, что давало путаницу.

## Что изменено

- Исправлен раздел `Сообщения` в админке:
  - восстановлена корректная обработка кнопок групп:
    - `Ожидают`
    - `Одобренные`
    - `Забаненные`
    - `Все`
  - добавлена “устойчивая” обработка кнопок рассылки:
    - `Добавить chat_id`
    - `Далее к тексту`
  - теперь поток рассылки продолжает работу даже если временно потерян `admin_mode` (после рестарта процесса/сброса контекста).

- Улучшен доступ для роли `banned`:
  - добавлено отдельное меню с двумя действиями:
    - `Статус`
    - `Вопросы / Поддержка`
  - исключено отображение стандартного пользовательского меню для заблокированных.

- Добавлено хранение причины блокировки:
  - новая таблица `user_blocks` в SQLite;
  - при бане сохраняются:
    - `reason`
    - `actor_chat_id`
    - `blocked_at`
  - данные используются для ответа в разделе `Статус`.

- Улучшен формат статуса заблокированного:
  - дата блокировки приводится к формату `dd-mm-YYYY HH:MM:SS`;
  - время отображается в локальной TZ сервера.

## Практический эффект

- Рассылки в админке снова стабильны и предсказуемы в эксплуатации.
- У заблокированных пользователей теперь понятный и целевой UX без лишних недоступных кнопок.
- Администратор и пользователь видят корректную причину и локальное время блокировки.
