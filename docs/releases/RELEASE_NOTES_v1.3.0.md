# WG Hope Bot v1.3.0

Фокус релиза: стабилизация production после reboot, real-time мониторинг и доработка UX админки/пользовательских разделов.

## Кратко

Релиз закрывает операционные риски, связанные с ручными изменениями uplink-интерфейсов и перезапусками сервера, и одновременно делает веб-морду и админку заметно удобнее в ежедневной эксплуатации.

## Что изменено

- Post-reboot автопроверка (новый контур надёжности):
  - добавлен `deploy/scripts/post_reboot_healthcheck.py`;
  - проверяются: `default route`, managed `ip rule`, `wg0`, `wg-hope-bot`, `wg-hope-monitor`, SSH, uplink-интерфейсы и регионы;
  - выполняется авто-очистка устаревших записей health и повторная синхронизация policy routing;
  - формируется подробный отчёт на русском с итогами `ОК/ИСПРАВЛЕНО/ПРЕДУПРЕЖДЕНИЕ/ОШИБКА`;
  - отчёт отправляется администраторам в Telegram.

- Автозапуск проверок после reboot:
  - добавлены:
    - `deploy/systemd/wg-hope-postboot-check.service`
    - `deploy/systemd/wg-hope-postboot-check.timer`
  - таймер запускает проверку дважды: через 2 и 5 минут после загрузки.

- Ручной запуск проверки из бота:
  - в `Админка -> Логи` добавлена кнопка `Ручной тест`;
  - отчёт возвращается в чат админа, длинный вывод отправляется частями.

- Безопасность uplink lifecycle:
  - усилен preflight добавления VPN-uplink (`amneziawg`, `wireguard`);
  - обязательно наличие `Table = off` в `[Interface]`;
  - при добавлении нового интерфейса создаётся безопасный stub-конфиг с `Table = off`.

- Пользовательский раздел `Информация`:
  - заменена старая одиночная кнопка `Инструкция`;
  - добавлено меню:
    - `Инструкция`
    - `Регионы`
    - `О проекте`
    - `WireGuard`
  - тексты `Регионы`, `О проекте`, `WireGuard` стали кастомизируемыми из админки.

- Админка: UX и читаемость:
  - в сценариях ввода включена клавиатура отмены `Назад`;
  - списки `Заявки/Активные/Баны/Администраторы` получили более понятный формат;
  - отчёт `Синхронизировать профили` стал подробным (какие пользователи с ошибкой и почему);
  - устранён конфликт кнопок `Регионы` в серверном меню (`Список регионов`).

- Веб-морда: real-time + новые блоки:
  - главная:
    - поиск и фильтры по пользователям;
    - новые KPI-карточки;
    - блоки `Новые заявки` и `Нагрузка сервера`;
  - добавлено live-обновление:
    - RX/TX и скорости на главной;
    - RX/TX и скорости на странице пользователя;
    - RX/TX, скорости, ping, статус и графики на вкладке `Серверы`;
  - добавлены real-time API:
    - `/api/realtime/dashboard`
    - `/api/realtime/servers`
    - `/api/realtime/user/{chat_id}`
    - `/api/realtime/user/{chat_id}/charts`
  - исправлен парсинг legacy timestamp (`... MSK`) и унифицирован вывод времени по локальной TZ сервера;
  - `Топ регионов` теперь показывает и новые регионы с нулевыми значениями.

- Hygiene репозитория:
  - расширен `.gitignore`, чтобы исключить из git рабочие БД/логи/бэкапы (`wg-bot.db`, `data.sqlite`, `*.db-wal`, `*.db-shm`, `chat/`, `backups/`, `*.bak*`).

## Практический эффект

- После reboot быстрее выявляются и автоматически исправляются типовые сетевые проблемы.
- Появился стандартный ручной smoke-тест прямо из бота.
- Риск повторения инцидентов с uplink-конфигами снижен за счёт жёсткого preflight.
- Операционная работа в админке и веб-морде стала заметно быстрее: меньше ручного поиска и больше live-данных.
