# 🚀 WG Hope Bot

WG Hope Bot - Telegram-бот для управления доступом к WireGuard VPN: выдача клиентских конфигов, роли, админка, логи и статистика.

## 📌 Релизы

- `v0.0.1` - историческая версия проекта
- `v1.0.0` - первый большой стабильный релиз новой архитектуры

## ✨ Возможности

- 👤 Роли пользователей: `super_owner`, `admin`, `user`, `pending`, `banned`
- 🔐 Управление доступом: заявки, одобрение, бан
- 🧩 Создание, отправка и отзыв WireGuard-конфигов
- 🖼️ Выдача `.conf` и QR-кода
- 📊 Статистика по пользователям, конфигам и IP
- 📝 Логи действий админов и пользователей
- 📣 Рассылки по выбранным chat_id или всем активным пользователям
- 🎚️ Гибкие лимиты числа конфигов на пользователя

## 🧱 Структура проекта

- `vpn_bot/main.py` - запуск приложения
- `vpn_bot/handlers.py` - Telegram-логика и сценарии
- `vpn_bot/actions.py` - операции с клиентами и файлами
- `vpn_bot/wireguard.py` - работа с `wg/wg-quick`, управление peer
- `vpn_bot/db.py` - SQLite, роли, клиенты, логи, статистика
- `vpn_bot/keyboards.py` - клавиатуры и inline-меню
- `vpn_bot/settings.py` - конфигурация через переменные окружения

## ⚙️ Требования

- Linux-сервер с установленными `wg` и `wg-quick`
- Доступ бота на запись в `WG_CONF` (обычно `/etc/wireguard/wg0.conf`)
- Настроенный WireGuard-интерфейс

## ✅ Текущее состояние

- База SQLite и логирование работают

## ⚠️ Важные условия

- Для операций с клиентами WireGuard должен быть установлен и доступен в PATH
- Для тестирования интерфейса без WireGuard можно запускать бот, но создание клиентов будет недоступно
- Все параметры задаются через `.env` на основе `.env.example`

## ▶️ Быстрый запуск (локально)

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# заполни BOT_TOKEN, SUPER_OWNER_CHAT_ID, SERVER_ENDPOINT, SERVER_PUBLIC_KEY
./run.sh
```

## 🐳 Запуск через Docker

```bash
docker compose up -d --build
docker logs -f wg-bot
```
