# Changelog

## v1.3.4

- Стабилизация маршрутизации при смене региона клиентом без массовых микродропов у остальных пользователей.
- Добавлен точечный apply маршрута для одного клиента:
  - новый путь применения только для IP конкретного клиента (`apply_client_egress_route`);
  - удаление только правил этого клиента при отзыве/удалении (`remove_client_egress_route`).
- Глобальный `full reconcile` сохранён для топологических изменений (интерфейсы/регионы/uplink-health), но исключён из обычного пользовательского region switch.
- Добавлена защита от гонок при операциях маршрутизации:
  - process lock + file lock (`/tmp/wg-hope-routing.lock`) для последовательного применения правил.
- Улучшен UX и предсказуемость:
  - повторный выбор того же региона теперь no-op (без лишних сетевых перестроек);
  - создание/удаление клиентского конфига больше не инициирует глобальную пересборку правил.

## v1.3.0

- Добавлена post-reboot автопроверка состояния сервера и VPN-контура:
  - новый скрипт `deploy/scripts/post_reboot_healthcheck.py` с проверками `default route`, managed `ip rule`, `wg0`, ключевых systemd-сервисов, SSH и ресурсов сервера;
  - проверка uplink-интерфейсов/регионов, авто-очистка устаревших health-записей, повторная синхронизация policy routing;
  - подробный отчёт на русском языке с итогами `ОК/ИСПРАВЛЕНО/ПРЕДУПРЕЖДЕНИЕ/ОШИБКА` и отправкой в Telegram админам;
  - добавлены systemd unit/timer: запуск через 2 и 5 минут после reboot.
- Добавлен ручной запуск post-reboot теста из Telegram-бота:
  - в `Админка -> Логи` добавлена кнопка `Ручной тест`;
  - запуск выполняется через `asyncio.to_thread`, вывод возвращается администратору чанками;
  - событие ручной проверки фиксируется в audit-логах (`manual_postboot_check`).
- Улучшен пользовательский раздел информации:
  - кнопка `Инструкция` заменена на `Информация` с подменю `Инструкция`, `Регионы`, `О проекте`, `WireGuard`;
  - добавлены новые кастомизируемые тексты в БД (`regions_text`, `about_text`, `wireguard_text`);
  - в админке (`Кастомизация`) добавлено редактирование всех новых разделов.
- Доработан UX админки:
  - в сценариях ввода активирована отдельная клавиатура отмены (`Назад`) вместо старых меню;
  - улучшено форматирование списков пользователей в `Заявки/Активные/Баны/Администраторы`;
  - в отчёте `Синхронизировать профили` добавлены детали ошибок по пользователям (`chat_id`, `@username`, причина);
  - устранён конфликт кнопки `Регионы` в разделе серверов (переименовано в `Список регионов`).
- Усилен preflight при добавлении uplink-интерфейсов:
  - для `amneziawg/wireguard` теперь жёстко проверяется наличие `Table = off` в `[Interface]`;
  - при первом добавлении интерфейса создаётся безопасный stub-конфиг с `Table = off`.
- Существенно расширен веб-мониторинг:
  - главная: добавлены карточки KPI, поиск и фильтры пользователей по нескольким критериям;
  - главная: добавлены блоки `Новые заявки` и `Нагрузка сервера` (CPU/RAM/Disk/Load/Uptime);
  - реализованы real-time API:
    - `/api/realtime/dashboard`
    - `/api/realtime/servers`
    - `/api/realtime/user/{chat_id}`
    - `/api/realtime/user/{chat_id}/charts`
  - реализовано live-обновление RX/TX, скоростей и графиков на главной, на странице пользователя и на вкладке `Серверы`;
  - исправлена обработка timestamp в старом формате (`YYYY-mm-dd HH:MM:SS MSK`) и приведён вывод к локальному времени сервера;
  - `Топ регионов` теперь учитывает даже регионы с нулевым количеством конфигов.
- Усилен hygiene репозитория:
  - расширен `.gitignore` для исключения рабочих БД/журналов (`wg-bot.db`, `data.sqlite`, `*.db-wal`, `*.db-shm`, `chat/`, `backups/`, `*.bak*`).

## v1.2.3

- Усилен production-контур для динамических uplink интерфейсов и регионов:
  - добавлена принудительная нормализация `Table = off` при замене VPN-uplink конфигов, чтобы исключить перехват default route;
  - добавлена синхронизация uplink-сервисов с systemd при старте бота (`enable/start` для включённых интерфейсов, `stop/disable` для выключенных);
  - включённые uplink-интерфейсы теперь поднимаются сразу и корректно переживают reboot.
- Улучшена безопасность маршрутизации и регионов:
  - маршрутизация больше не использует выключенные uplink интерфейсы (`enabled=0`);
  - назначение региона запрещено для выключенных интерфейсов и неготовых VPN-uplink;
  - проверка интерфейсов в админке дополнена stale-handshake probe (`probe=ok|fail`) для уменьшения ложных статусов.
- Улучшена надёжность мониторинга и восстановления:
  - улучшено поведение down-alert после рестарта для ранее упавших uplink (`UPLINK_ALERT_DOWN_ON_START`, включён по умолчанию);
  - исправлены переходы состояния алертов: recovery корректно фиксирует `last_alert_state=ok`.
- Консолидированы сетевые/production обновления:
  - дефолтная подсеть обновлена до `VPN_SUBNET=10.8.0.0/22` в настройках и шаблонах;
  - systemd-запуск бота обновлён на прямой вызов `bot.py` для более чистого старта.
- Результат по стабильности:
  - подтверждена устойчивость после повторных reboot: автоподъём uplink, сохранение policy routing и отсутствие нежелательного fallback в другой регион при доступном целевом uplink.

## v1.2.0

  - healthcheck по интерфейсам;

- Существенно расширен веб-мониторинг:
